## Session 1 fixes
- `env.staging.example` rewritten — was missing COMPOSE_PROJECT_NAME, GATEWAY_PORT, NGINX_PORT, FRONTEND_PORT, *_IMAGE, RABBITMQ_*, SPRING_PROFILE
- Gateway `show-details: always` → `when-authorized` (security: health details were public)
- Gateway CORS hardcoded origins → `${CORS_ORIGINS}` env var; `CORS_ORIGINS` added to docker-compose gateway service
- Frontend CI `pnpm typecheck || true` → `pnpm typecheck` (TypeScript errors were silently ignored)
- CI docker jobs: removed unused "Download JAR" steps (Dockerfile rebuilds from source, downloaded artifact was ignored)

## Session 2 fixes
- **Critical auth bug fixed**: Gateway `JwtService` used `UTF-8 bytes` key derivation, monolith uses `Base64 decode`. Tokens generated by monolith could NOT be validated by gateway in production.
- `hrms-common JwtService` updated: added `isValid`, `extractUserId`, `extractEmail`, `extractRole` methods; added default 0 values to `@Value` expiry fields so gateway (validation-only) doesn't fail on startup; added `spring-security-core:provided` to pom.xml
- `api-gateway` now depends on `hrms-common` instead of duplicating JwtService; removed redundant jjwt deps; `ApiGatewayApplication` expanded `scanBasePackages` to include `kz.aitu.hrms.common.security`; `JwtAuthFilter` import updated; duplicate `security/JwtService.java` deleted
- `hrms-common-publish.yml` created — publishes to GitHub Packages Maven registry on push to main/develop (requires updating OWNER placeholder in pom.xml distributionManagement URL)
- `gateway-ci.yml`: added `services/hrms-common/**` to path triggers; added `Install hrms-common` step before `mvn verify`
- All 4 workflow files: replaced hardcoded `http://192.168.100.53:8091` with `${{ vars.STAGING_URL }}` (GitHub Actions Variable — must be set in repo/org settings)
- `_service-template.yml`: removed unused Download JAR step

## Remaining known issues (not yet fixed)
- `docker/docker-compose.prod.yml` and `docker/docker-compose.staging.yml` are orphaned legacy files
- Prometheus config uses `host.docker.internal` (only works on Docker Desktop, not Linux)
- `hrms-common` `distributionManagement` URL has `OWNER` placeholder — must be updated before `mvn deploy` works
- `STAGING_URL` GitHub Actions Variable must be added in repo settings

**Why:** Prevent silent auth failures in production and improve CI reliability.
**How to apply:** When touching JWT, auth, or CI workflows, be aware of the Base64 key derivation requirement and the hrms-common dependency chain.
